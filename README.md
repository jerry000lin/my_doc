# 个人复习文档

## 定位

这个仓库用于沉淀可复用的技术复习资料、工程实践笔记、脚本工具说明和业务方法论。内容应当能脱离聊天上下文阅读，适合作为个人知识库、面试复盘材料或项目实践参考。

## 目录说明

| 目录 | 内容边界 |
| --- | --- |
| `前端/` | 前端框架、工程化、类型系统、运行机制和示例代码。 |
| `大数据/` | Spark、PySpark、数据开发脚本、数据平台相关实践。 |
| `风控/` | 风控建模、样本定义、因子筛选、策略分析和业务方法论。 |
| `个人工具/` | 面向个人效率和浏览器增强的脚本、配置和使用说明。 |
| `python/` | Python 语言、工程实践和常用库复习资料。 |

## 维护规则

- 文档优先放在对应技术域或业务域下，不在根目录堆放专题文章。
- 可执行脚本放入 `scripts/`、`tools/` 或具体工具目录，避免和文档混放。
- 示例代码放入 `examples/`，文件名使用英文、下划线或短横线，便于跨平台处理。
- 系列文档使用两位编号，例如 `01-主题.md`、`02-主题.md`。
- 缓存、编译产物、临时结果文件不进入知识库结构。

## 推荐阅读路径

前端方向可以从 `前端/react/` 和 `前端/typescript/` 开始，重点关注状态边界、类型建模和工程化机制。

大数据方向可以从 `大数据/spark/` 开始，再结合 `大数据/scripts/` 中的脚本理解实际数据访问和排查流程。

风控方向可以从 `风控/样本定义/` 开始，先明确样本边界，再扩展到因子筛选和策略评估。
