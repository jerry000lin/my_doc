# 风控样本定义（Labeling）方法论：业务目标、时间结构与样本分层的统一框架

## 核心定义：风控样本定义的本质与四个前置业务判定

### 样本定义的本质

风控样本定义的本质是**业务决策目标的数学化表达**。模型训练所使用的好样本、坏样本与灰样本，对应某个业务动作在某个时间点之前所需的风险判断。样本定义的优劣，取决于它是否与**业务动作**、**数据可得性**、**时间成熟度**和**结果解释口径**保持一致。

### 前置判定一：明确作用点

模型服务的是申请前准入、授信前排序、存量经营、贷中预警、催收分案，还是复贷经营。**作用点决定样本观察时点**，也决定标签应当刻画**未来表现**、**当前状态**还是**历史风险痕迹**。

### 前置判定二：明确优化目标

风险管理中的常见目标包括**短期稳定性**、**强坏风险识别**、**中长期资产质量**、**当前可经营性**和**组合扩量效率**。目标不同，**坏样本阈值**、**观察窗口**和**灰样本边界**都会变化。短期稳定性偏向 **30+ / 60+** 的短窗口标签，中长期质量偏向 **12M** 及以上的固定观察窗，当前可经营性偏向**当前状态标签**或**状态分层**。

### 前置判定三：明确结果用途

模型输出用于**硬拒绝**、**授信额度决策**、**营销名单排序**、**策略分层**、**人群圈选**还是**单纯的因子验证**。硬决策场景要求标签边界严格、误伤成本可控；排序场景允许边界更宽，只要分数具有稳定的单调区分能力即可；因子验证场景优先保证样本纯度。

### 前置判定四：明确口径归属

标签可以是**标准风险标签**，也可以是**代理标签**。标准标签用于正式风控管理，强调**固定观察窗**、**时间对称**和**审计可追溯性**；代理标签用于当前阶段建模验证、因子筛选和策略探索，强调**业务可解释性**与**样本可得性**。两者的**坏样本率**、**时间窗口**和**管理含义**需要明确区分。

这四个判定决定后续标签结构。**业务动作未定义，标签无法收敛；结果用途未定义，样本边界会持续漂移；代理标签与标准标签未区分，后续沟通会持续出现口径冲突。**

## 关键维度构建：主体、观察时点、时间窗口的设计机制

### 样本主体：标签附着在哪个对象上

样本主体是标签附着的最小业务对象，常见有**合同**、**借据**、**申请**、**客户**、**客户-月份**、**客户-申请时点**、**客户-渠道-月份**等。主体选择需要与**业务动作**和**特征粒度**一致。

当模型服务于某一笔申请的审批或授信，主体应接近**申请**或**申请时点**。当模型服务于客户经营、名单生成或经营打分，主体更适合是**客户-月份**或**客户-策略时点**。当模型服务于贷中监控和滚动预警，主体更适合是**存量合同-月末时点**。

样本主体的设计机制可以概括为：**决策粒度决定标签粒度，标签粒度决定后续特征对齐方式**。如果后续特征只能按 **`party_id + feature_dt`** 提供，最终样本主体最好也落在 **`party_id + feature_dt`**。如果主体与特征粒度不一致，后续建模会出现**多对一聚合**、**标签重复**、**同一客户多样本冲突**等问题。

### 观察时点：模型站在什么时候看客户

观察时点是模型允许使用信息的截止点。风控模型的输入需要严格限定在**观察时点及其之前**；标签需要来自**观察时点之后**。观察时点的常见形式包括**申请日前一个月末**、**授信前一日**、**放款当日**、**月末快照日**、**当前最新时点**。

观察时点的设计机制可以写成一个固定公式：

> **观察时点 = 业务动作发生之前、业务系统真实可见、特征能够稳定落地的最后一个时点**

例如，申请前准入模型通常取**申请前一个月末**；存量经营模型通常取**当前月末**；贷中预警模型通常取**当前账龄状态形成之后、下一个风险迁徙周期之前**的某一管理时点。观察时点一旦确定，所有特征都必须服从这个时间边界。任何将**申请后**、**放款后**、**催收后**信息混入特征的做法，都会造成**数据泄漏**。

### 时间窗口：标签观察范围如何推导

时间窗口的选择本质上是**风险目标的设定**。

短窗口用于识别**早期风险**。典型形式是放款后 **3M / 6M** 内是否出现 **30+**、**60+**。这类标签适合准入前筛掉“放款后很快出险”的客户，适用于**营销前置风控**、**快速放量**、**早期资产质量控制**。它的优点是**成熟快**、**样本量足**、**对短期策略更敏感**；它的代价是会把窗口外才发生的风险视为负样本。

长窗口用于识别**中长期质量**。典型形式是放款后 **12M** 内是否达到 **30+ / 60+**，或者更长的生命周期风险。它的优点是**风险定义更完整**，更接近资产全周期质量；它的代价是**未成熟样本更多**、**训练数据更新更慢**、**第一版项目推进速度更低**。

当前状态窗口用于**经营和存量管理**。它不看固定未来，而看“**当前是否已坏**”“**当前是否正常**”“**当前正常但历史是否脏过**”。这类标签特别适合**当前名单经营**、**存量分层**、**代理样本构造**，但不适合作为正式固定观察窗风险标签的直接替代。

窗口设计的推导逻辑如下：

* 目标是**短期稳定性**，优先**短窗口**；
* 目标是**长期质量**，优先**长窗口**；
* 目标是**当前可经营性**，优先**当前状态**；
* 目标是**当前阶段快速验证因子或策略**，允许使用**当前状态 + 历史痕迹**的代理样本，但需要清楚标记其**代理性质**。

## Labeling 策略选型矩阵：好 / 坏 / 灰样本的界定标准与排查逻辑

### 坏样本：风险定义的锚点

坏样本定义承载两层信息：**风险强度**和**业务关切**。常见定义包括**当前 30+**、**当前 60+**、**固定窗口内 ever30 / ever60**、**核销**、**拒绝**、**资产损失**等。不同坏样本定义对应不同业务语义。

* **30+** 表示较早显现的风险，样本更多，模型更容易训练，适合短期经营前筛客。
* **60+** 表示更强的坏，边界更稳，适合强坏识别和更谨慎的风控策略。
* **核销 / 损失** 表示最强坏，但样本稀疏、成熟慢。
* **当前坏** 表示“此刻已处于风险状态”，适合当前经营和当前存量治理。
* **固定窗口内的坏** 表示“未来一定时期内的结果”，适合标准预测问题。

坏样本优先选择**边界清晰**、**业务认可**、**后续容易验收**的定义。第一版模型更适合从**最明确的坏样本**开始。边界越模糊，后续越难解释模型分数。

### 好样本：样本纯度的核心来源

好样本定义比坏样本更难，因为坏样本通常有较强业务事件约束，好样本则容易把大量“**不是坏，但也不够好**”的中间层混入其中。好样本的构建原则是：**先定义最纯净的好，再决定是否扩大边界**。

常见好样本定义包括：

* **固定窗口内从未达到坏阈值**；
* **当前正常且历史从未逾期**；
* **放款满 6M / 12M 后仍保持当前 0 且历史干净**；
* **当前正常但历史曾有轻微逾期**；
* **当前正常但历史上曾有严重逾期后恢复**。

这些对象不能混在一起。**当前正常且历史干净**代表最稳定的好样本；**当前正常但历史有逾期**属于可经营但并不纯净的对象；**未成熟样本**只说明风险尚未充分暴露，不能直接当作好样本。

### 灰样本：边界管理的工具

灰样本用于承接不适合直接归为 **0 / 1** 的中间层，常见有以下几类：

* **观察期未成熟**；
* **当前轻微逾期但未达到坏阈值**；
* **当前正常但历史存在逾期痕迹**；
* **数据口径不稳定、主体重复、边界模糊**；
* **当前阶段不纳入训练但值得保留做后续策略分析的样本**。

**灰样本是对不确定性的显式管理。** 第一版代理模型中，把中间层先归灰，往往比强行压成 **0 / 1** 更稳。

### 排查逻辑：标签自洽性的验证顺序

标签定义完成之后，需要做四类基础排查：

* **逻辑自洽性**：`Y=1` 是否一定满足坏条件；`Y=0` 是否一定满足好条件；`Y=null` 是否只来自灰样本；
* **时间成熟度**：最近月份是否因为观察期未满而缺失 **0** 样本；坏率是否因成熟度差异被抬高；
* **主体稳定性**：同一客户同一观察时点下是否存在多笔样本；主体聚合后是否引入标签冲突；
* **映射覆盖率**：训练主体主键是否能稳定映射到后续特征主键；缺失比例是否可接受。

这一轮排查是 **Labeling 成功与否的正式验证环节**。

## 三大演进路线对比：固定窗口 vs 当前状态 vs 历史痕迹分层

### 固定观察窗标签

固定观察窗标签的典型形式是：

* 放款后 **6M** 内是否 **ever30 / ever60**
* 放款后 **12M** 内是否 **ever30 / ever60**

这一路线适合**标准准入模型**、**审批模型**、**授信模型**和需要**审计追溯**的正式风控标签体系。它的优点是**时间窗口对称**、**样本定义严谨**、**验证体系清晰**；它的代价是需要更高的数据完整性和更长的样本成熟时间。窗口外发生的风险不会进入正样本，因此它优化的是**指定窗口内的风险**，不是全生命周期风险。

当业务问题是“申请前如何判断这笔未来贷款在前 **6M / 12M** 内是否会出现明显坏账表现”，固定窗口标签是最合适的表达方式。

### 当前状态标签

当前状态标签的典型形式是：

* 当前 **`dpd = 0`**
* 当前 **`1~30`**
* 当前 **`30+`**
* 当前 **`60+`**

这一路线适合**当前经营**、**当前名单治理**、**存量分层**、**快速代理样本构造**。它的优点是**业务直觉强**、**沟通成本低**、**容易与现有经营口径对齐**；它的代价是无法完整表达历史风险，也无法单独代表长期信用质量。历史上坏过但现在恢复的人，在当前状态标签下会被视为当前正常。

当业务问题是“现在要不要经营这个人”“现在哪些客户可以作为可经营客群”，当前状态标签有很强实用性。

### 当前状态 + 历史痕迹分层

这一路线是当前状态标签的强化版。它保留**当前状态**，同时引入**历史风险痕迹**和**成熟度信息**，把样本划分为：

* **当前坏**；
* **当前好且历史干净**；
* **当前好但历史有逾期记录**；
* **未成熟**；
* **轻微风险**；
* **其他边界对象**。

这一路线最适合**代理样本场景**：真实标准标签暂时不完备，但项目又需要先推进模型、打分或策略验证。它的优点是兼顾**业务可解释性**、**负样本纯度**和**样本可得性**；它的代价是标签不再是标准固定观察窗，不适合被解释成“真实违约率”。

### 三条路线的选择逻辑

* **固定窗口**适合标准预测问题，强调**未来结果**；
* **当前状态**适合当前经营问题，强调**现在能否用**；
* **历史痕迹分层**适合代理样本阶段，强调**当前可落地且样本纯度可控**。

真正的选择标准是**业务动作**：

* 问题是“未来 **6M** 会不会快坏”，固定窗口优先；
* 问题是“当前哪些人不能经营”，当前状态优先；
* 问题是“先构出一版稳定代理样本跑模型”，历史痕迹分层优先。

## 常见误区与避坑指南：错误示例与修正方案

### 误区一：把“非坏样本”直接等同于“好样本”

错误示例：只要未达到 **30+ / 60+**，一律记为 **0**。
修正方案：先定义最纯净的 **0**，把**轻度逾期**、**历史有脏记录**、**未成熟样本**全部打灰。第一版模型优先学习“最干净的好样本”和“最明确的坏样本”。

### 误区二：混用当前状态、历史 ever 和固定观察窗

错误示例：正样本用“当前 **30+**”，负样本用“历史从未逾期”，窗口逻辑互不对称。
修正方案：先定义标签所属路线。**固定观察窗**、**当前状态**、**历史痕迹分层**三类口径需要单独成体系，不能在一个标签里随意混搭。

### 误区三：忽略成熟度，直接比较最近月份的坏率

错误示例：最近月份几乎没有 **0** 样本，却直接拿来与历史月份做坏率对比。
修正方案：对存在**未成熟逻辑**的标签，必须做**时间截断**；否则近端月份会因为“好样本尚未长成”而出现虚高坏率。

### 误区四：把代理样本的正样本占比直接当作真实违约率

错误示例：把当前坏 / 当前好且历史干净构造出的 **`y=1 / y=0`** 比例直接解释为业务坏账率。
修正方案：明确写明“**代理坏样本占比**”或“**代理正样本占比**”，不直接对外表述为真实违约率。

### 误区五：在代理样本阶段过度解读产品、渠道、年份结构

错误示例：在代理样本层面直接输出复杂产品分层结论，并尝试以此替代业务认知。
修正方案：细分结构排查仅作为**内部理解数据结构与样本组成**的背景信息。对外主线围绕**样本定义**、**风险分层**与**模型区分能力**展开。

### 误区六：一开始就做多目标建模

错误示例：**30+、60+、6M、12M、当前坏**同时塞进一个模型目标体系。
修正方案：先做**规则分层**，再选一个最主目标做第一版二分类。多目标建模只有在**标签体系稳定**、**业务目标清晰**、**样本量充足**时才有意义。

### 误区七：样本主体和特征粒度不一致

错误示例：标签按合同定义，特征按客户月聚合，后续再临时拼接。
修正方案：先确定业务动作对应的主体粒度，再确定最终训练样本粒度。**`party_id + feature_dt`** 适合客户经营类模型，**申请级样本**适合审批类模型，**合同级样本**适合贷中预警类模型。

## 附录：风控样本定义通用 SOP 决策流

### Step 1：先确定业务动作

* **申请前准入**
* **授信前排序**
* **当前经营筛客**
* **存量分层**
* **贷中预警**
* **催收分案**

没有**业务动作**，标签无法定义。

### Step 2：确定观察时点

* **申请前一个月末**
* **放款日**
* **当前月末**
* **当前最新时点**

规则：**特征**只能来自**观察时点及之前**；**标签**只能来自**观察时点之后**。

### Step 3：确定目标类型

* **短期稳定性**：优先 **30+ / 60+ + 短窗口**
* **强坏识别**：优先 **60+ / 核销**
* **中长期质量**：优先 **12M+ 固定观察窗**
* **当前可经营性**：优先 **当前状态标签**
* **代理样本建模**：优先 **当前状态 + 历史痕迹分层**

### Step 4：选标签路线

#### 固定窗口

适合**标准准入 / 审批模型**
形式：放款后 **6M / 12M** 内是否 **ever30 / ever60**

#### 当前状态

适合**当前名单经营 / 存量治理**
形式：当前 **`0 / 1~30 / 30+ / 60+`**

#### 历史痕迹分层

适合**代理样本 / 当前阶段快速建模**
形式：**当前坏、当前好且历史干净、当前好但历史脏、未成熟 / 其他**

### Step 5：先定坏样本

优先定义**边界最清楚**、**业务最认可**的坏样本：

* **核销 / 损失**
* **60+**
* **30+**
* **当前坏**
* **固定窗口内坏**

第一版优先用**最明确的坏**。

### Step 6：再定最纯净的好样本

优先定义“**最干净的 0**”，不要先追求大规模 **0**。

常见纯净好样本：

* **当前 0 + 历史干净**
* **固定窗口内从未逾期**
* **放款满 6M / 12M 且从未逾期**

### Step 7：其余先灰掉

灰样本优先承接：

* **未成熟**
* **当前轻度逾期**
* **当前正常但历史脏**
* **口径边界模糊**
* **当前阶段不打算纳入训练的中间层**

### Step 8：做 4 类必要校验

1. **标签逻辑自洽**：**1 / 0 / null** 是否满足定义
2. **时间成熟度**：最近月份是否几乎没有 **0**
3. **主体稳定性**：同一主体同一时点是否多笔样本冲突
4. **主键覆盖率**：能否稳定映射到特征主键

### Step 9：决定训练样本

第一版模型优先用：

* **最明确的坏**
* **最纯净的好**

中间层先不进训练。

### Step 10：判断是否需要时间截断

如果有**未成熟逻辑**，训练集通常需要截断最近月份。
判断标准：最近月份是否出现 **`y0 ≈ 0`**、坏率异常抬高。

### Step 11：代理样本沟通口径

必须写清楚：

* 这是真实标签还是**代理标签**
* 这是**当前状态标签**还是**固定观察窗标签**
* 这个正样本占比是不是业务真实违约率

代理样本用于当前阶段的**建模、打分、因子验证或策略探索**。

### Step 12：模型策略

标签体系仍在探索期时：

* 先做**规则分层**
* 再做一个主目标**二分类模型**
* 需要时再加**辅助模型**
* 不优先做**多目标建模**

### 快速决策表

#### 业务问“现在哪些人不能经营”

选：**当前状态标签**

#### 业务问“放款后前 6M 会不会快坏”

选：**固定窗口 6M 标签**

#### 业务问“先拿现有数据做一版代理样本跑模型”

选：**当前状态 + 历史痕迹分层**

#### 业务问“这版坏率能不能直接当真实违约率”

回答：**不能**。代理标签只表示代理样本下的**正样本占比**。

### 训练样本最小可用模板

* **作用点**：明确
* **主体**：明确
* **观察时点**：明确
* **坏样本**：明确
* **好样本**：纯净
* **灰样本**：显式定义
* **成熟度**：检查通过
* **时间截断**：必要时执行
* **特征主键映射**：通过
* **标签口径说明**：可对内对外解释

这套流程完成后，再进入**特征接入、模型训练与分箱分析**。
